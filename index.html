Syntax help & supported types
Prefix syntax (forces type, one per line):
  • ipv4=192.168.1.1 — IPv4 address or CIDR
  • ipv6=::1 — IPv6 address
  • ip=10.0.0.1 — auto v4/v6
  • domain=evil.com — FQDN / hostname
  • url=https://evil.com/path — full URL
  • md5= / sha1= / sha256= — hash
  • hash=abc123… — auto-detect hash length
  • path=C:\Windows\evil.exe — file path
  • file=evil.exe — file name only
  • cmdline=powershell -enc … — command line
  • registry=HKLM\… — registry key
  • email=attacker@evil.com
  • ua=Mozilla/5.0… — user agent
CSV upload: accepts these column layouts:
  • type, value (e.g. ipv4, 1.2.3.4)
  • value, type (reversed)
  • Single column — auto-detected
  • Optional header row is skipped automatically
Auto-detected types (no prefix needed):
  • IPv4/IPv6 — from structure
  • SHA256 (64 hex), SHA1 (40), MD5 (32)
  • URLs starting with http:// or https://
  • Email addresses
  • Registry keys starting with HKLM / HKEY_
  • Windows paths (C:\…, \\server\…)
  • Unix paths (/usr/…)
Ambiguous entries (e.g. evil.exe) will ask you to confirm the type.

Sigma Help Rule writing quick reference

Required Rule Shape

title: Suspicious PowerShell Encoded Command
id: 123e4567-e89b-12d3-a456-426614174000
status: experimental
description: Detects PowerShell launched with encoded command arguments
author: Your Team
logsource:
  product: windows
  category: process_creation
detection:
  selection:
    Image|endswith:
      - '\\powershell.exe'
    CommandLine|contains:
      - ' -enc '
  condition: selection
level: high

Build It Correctly

  • Always define logsource.product and logsource.category.
  • Put field logic under one or more detection blocks (for example: selection, filter_legit).
  • Use Sigma modifiers for matching behavior: |contains, |startswith, |endswith, |re.
  • Set condition explicitly, for example: selection and not filter_legit.
  • Pick a realistic level and include helpful metadata (tags, references, falsepositives).

Good Practices

  • Use specific fields first (for example Image + CommandLine) instead of broad keywords only.
  • Add a filter_* block for known benign noise.
  • Keep values normalized and escaped correctly.

Common Mistakes

  • Missing detection.condition.
  • Using SIEM query syntax inside Sigma YAML.
  • Forgetting YAML indentation consistency.
